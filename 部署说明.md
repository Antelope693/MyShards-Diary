# MyShards系统部署说明

## 📋 前置要求

1. **服务器要求**
   - Linux 服务器（推荐 Ubuntu 20.04+ 或 CentOS 7+）
   - 至少 2GB RAM
   - 至少 10GB 可用磁盘空间
   - 已安装 Docker 和 Docker Compose

2. **安装 Docker 和 Docker Compose**
   ```bash
   # Ubuntu/Debian
   curl -fsSL https://get.docker.com -o get-docker.sh
   sudo sh get-docker.sh
   sudo usermod -aG docker $USER
   
   # 安装 Docker Compose
   sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
   sudo chmod +x /usr/local/bin/docker-compose
   ```

## 🚀 快速部署步骤

### 1. 上传项目文件

将项目文件上传到服务器，例如：
```bash
# 使用 scp 上传
scp -r Diary/ user@your-server:/opt/diary

# 或使用 git
git clone your-repo-url /opt/diary
cd /opt/diary
```

### 2. 创建配置文件

```bash
# 复制配置模板
cp env.example .env

# 编辑配置文件
nano .env  # 或使用 vim/vi
```

### 3. 填写配置信息

在 `.env` 文件中填写以下必要信息：

```env
# 管理员密码（必须修改！）
ADMIN_PASSWORD=your_secure_password_here

# 服务器域名或IP
SERVER_DOMAIN=your-domain.com

# 端口配置（默认即可）
FRONTEND_PORT=80
BACKEND_PORT=3001
```

**重要配置说明：**

- `ADMIN_PASSWORD`: 管理员登录密码，请务必设置为强密码
- `SERVER_DOMAIN`: 
  - 如果使用域名：填写你的域名，如 `diary.example.com`
  - 如果使用IP：填写服务器IP，如 `192.168.1.100`
  - 本地测试：填写 `localhost`

### 4. 运行部署脚本

```bash
# 给脚本添加执行权限
chmod +x deploy.sh

# 运行部署脚本
./deploy.sh
```

或者手动部署：

```bash
# 构建并启动服务
docker-compose -f docker-compose.prod.yml up -d --build
```

### 5. 验证部署

```bash
# 查看服务状态
docker-compose -f docker-compose.prod.yml ps

# 查看日志
docker-compose -f docker-compose.prod.yml logs -f
```

访问 `http://your-domain.com` 或 `http://your-server-ip` 查看网站。

## 🔧 常用管理命令

### 查看服务状态
```bash

```

```bash

```

```bash

docker-compose -f docker-compose.prod.yml up -d
```

## 🔒 安全配置

### 1. 防火墙设置

```bash
# Ubuntu/Debian (UFW)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp  # 如果使用HTTPS
sudo ufw enable

# CentOS (firewalld)
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --reload
```

### 2. 配置 HTTPS（推荐）

使用 Nginx 反向代理 + Let's Encrypt：

```bash
# 安装 Certbot
sudo apt-get update
sudo apt-get install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com
```

### 3. 修改默认端口（可选）

如果不想使用 80 端口，可以修改 `.env` 文件：
```env
FRONTEND_PORT=8080
```

然后访问 `http://your-domain.com:8080`

## 🐛 故障排除

### 1. 服务无法启动

```bash
# 查看详细错误信息
docker-compose -f docker-compose.prod.yml logs

# 检查端口是否被占用
sudo netstat -tulpn | grep :80
sudo netstat -tulpn | grep :3001
```

### 2. 无法访问网站

- 检查防火墙设置
- 检查 Docker 容器是否运行：`docker ps`
- 检查端口是否正确映射
- 查看 Nginx 日志：`docker-compose -f docker-compose.prod.yml logs frontend`

### 3. 数据库问题

```bash
# 检查数据库文件权限
ls -la backend/data/

# 如果需要重置数据库（会删除所有数据！）
rm -rf backend/data/*
docker-compose -f docker-compose.prod.yml restart backend
```

### 4. 图片无法显示

```bash
# 检查上传目录权限
ls -la backend/uploads/

# 确保目录有写权限
chmod -R 755 backend/uploads/
```

## 📝 注意事项

1. **定期备份**：建议每天备份 `backend/data` 和 `backend/uploads` 目录
2. **密码安全**：定期更换管理员密码
3. **更新维护**：定期更新 Docker 镜像和系统
4. **日志管理**：定期清理日志文件，避免占用过多磁盘空间

## 🆘 获取帮助

如果遇到问题，请检查：
1. Docker 和 Docker Compose 是否正确安装
2. 配置文件 `.env` 是否正确填写
3. 端口是否被其他程序占用
4. 防火墙是否允许相应端口

查看详细日志：
```bash
docker-compose -f docker-compose.prod.yml logs --tail=100
```
