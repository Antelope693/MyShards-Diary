# 🚀 服务器部署步骤

代码已推送到 GitHub，现在可以部署到服务器了。

## 📋 部署前准备

### 1. 确保代码已推送

如果推送时遇到认证问题，可以：

**方法一：使用 Personal Access Token**
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
2. 生成新 token，勾选 `repo` 权限
3. 推送时使用 token 作为密码

**方法二：使用 SSH（推荐）**
```bash
# 更改远程地址为 SSH
git remote set-url origin git@github.com:Antelope693/MyShards-Diary.git
```

### 2. 服务器要求

- Linux 服务器（Ubuntu 20.04+ 推荐）
- 已安装 Docker 和 Docker Compose
- 有公网 IP 或域名
- 开放 80 端口（或你配置的其他端口）

---

## 🔧 服务器端部署步骤

### 步骤 1: 连接到服务器

```bash
ssh root@你的服务器IP
```

### 步骤 2: 安装 Docker（如果未安装）

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
systemctl start docker
systemctl enable docker

# 安装 Docker Compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
```

### 步骤 3: 克隆项目

```bash
# 进入合适的目录
cd /opt

# 克隆项目
git clone https://github.com/Antelope693/MyShards-Diary.git diary
cd diary
```

### 步骤 4: 创建配置文件

```bash
# 复制配置模板
cp env.example .env

# 编辑配置文件
nano .env
```

**必须填写的内容：**

```env
# 管理员密码（必须修改！）
ADMIN_PASSWORD=你的强密码

# 服务器域名或IP（必须填写！）
SERVER_DOMAIN=你的域名或IP
# 例如：
# SERVER_DOMAIN=diary.example.com
# 或
# SERVER_DOMAIN=192.168.1.100
```

保存文件：`Ctrl + O` → `Enter` → `Ctrl + X`

### 步骤 5: 部署应用

```bash
# 给部署脚本添加执行权限
chmod +x deploy.sh

# 运行部署脚本
./deploy.sh
```

或者手动部署：

```bash
docker-compose -f docker-compose.prod.yml up -d --build
```

### 步骤 6: 检查服务状态

```bash
# 查看服务状态
docker-compose -f docker-compose.prod.yml ps

# 查看日志
docker-compose -f docker-compose.prod.yml logs -f
```

按 `Ctrl + C` 退出日志查看。

### 步骤 7: 配置防火墙（如果需要）

```bash
# Ubuntu/Debian
sudo ufw allow 80/tcp
sudo ufw enable

# CentOS
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --reload
```

## ✅ 验证部署

### 1. 检查服务

```bash
docker-compose -f docker-compose.prod.yml ps
```

应该看到两个服务都是 `Up` 状态。

### 2. 访问网站

在浏览器中访问：
- **前端地址**：`http://你的域名` 或 `http://你的IP`
- **管理页面**：`http://你的域名/admin` 或 `http://你的IP/admin`

### 3. 测试登录

1. 访问管理页面
2. 输入你在 `.env` 文件中设置的密码
3. 如果成功登录，说明部署成功！

## 🐛 常见问题

### 问题 1: 无法访问网站

**检查：**
1. 服务是否运行：`docker-compose -f docker-compose.prod.yml ps`
2. 防火墙是否开放端口：`sudo ufw status`
3. 端口是否被占用：`netstat -tulpn | grep :80`

### 问题 2: 登录失败

**检查：**
1. `.env` 文件中的密码是否正确
2. 重启后端服务：`docker-compose -f docker-compose.prod.yml restart backend`

### 问题 3: 构建失败

**解决：**
```bash
# 清理并重新构建
docker system prune -a
docker-compose -f docker-compose.prod.yml build --no-cache
```
