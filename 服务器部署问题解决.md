# 服务器部署问题解决指南

## 问题：sqlite3 编译失败

### 症状
```
npm error gyp ERR! find Python Python is not set from command line or npm configuration
npm error gyp ERR! find Python You need to install the latest version of Python.
```

### 原因
`sqlite3` 是一个原生 Node.js 模块，需要在构建时编译。Alpine Linux 镜像默认不包含编译所需的工具。

### 解决方案

已修复的 Dockerfile 包含：
- `sqlite` - SQLite 运行时库
- `sqlite-dev` - SQLite 开发库（头文件）
- `python3` - Python 解释器（node-gyp 需要）
- `make` - 构建工具
- `g++` - C++ 编译器

### 部署步骤

1. **拉取最新代码**：
   ```bash
   git pull
   ```

2. **清理旧的构建缓存**（可选，但推荐）：
   ```bash
   docker-compose -f docker-compose.prod.yml down
   docker system prune -f
   ```

3. **重新构建**：
   ```bash
   docker-compose -f docker-compose.prod.yml up -d --build
   ```

### 构建时间

- **首次构建**：可能需要 5-10 分钟（需要编译 sqlite3）
- **后续构建**：如果使用缓存，通常 1-2 分钟

### 如果构建仍然很慢

如果服务器在国内，可以尝试使用国内镜像源加速：

1. **修改 Dockerfile**（临时方案）：
   ```dockerfile
   # 在安装构建依赖之前添加
   RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.aliyun.com/g' /etc/apk/repositories
   ```

2. **或者使用代理**：
   ```bash
   # 设置代理环境变量
   export HTTP_PROXY=http://your-proxy:port
   export HTTPS_PROXY=http://your-proxy:port
   docker-compose -f docker-compose.prod.yml up -d --build
   ```

### 验证构建

构建完成后，检查服务状态：
```bash
docker-compose -f docker-compose.prod.yml ps
```

应该看到两个服务都是 `Up` 状态。

### 查看构建日志

如果构建失败，查看详细日志：
```bash
docker-compose -f docker-compose.prod.yml build --no-cache backend 2>&1 | tee build.log
```

### 常见问题

#### 1. 构建超时
如果构建时间超过 30 分钟，可能是网络问题。建议：
- 检查网络连接
- 使用镜像源加速
- 考虑使用代理

#### 2. 内存不足
编译 sqlite3 需要一定内存。如果服务器内存不足：
- 增加交换空间（swap）
- 或者使用预编译的二进制文件（需要修改 package.json）

#### 3. 权限问题
确保有足够的权限：
```bash
# 检查 Docker 权限
docker ps

# 如果提示权限不足，可能需要：
sudo usermod -aG docker $USER
# 然后重新登录
```
