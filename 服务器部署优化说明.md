# 服务器部署优化说明

## 问题：依赖下载超时

如果构建时 `apk add` 步骤耗时过长（超过 10 分钟），通常是网络问题。

### 解决方案 1：使用国内镜像源（推荐，如果服务器在国内）

修改 `backend/Dockerfile`，在安装依赖之前添加镜像源：

```dockerfile
FROM node:20-alpine

WORKDIR /app

# 使用国内镜像源加速
RUN sed -i 's/dl-cdn.alpinelinux.org/mirrors.aliyun.com/g' /etc/apk/repositories

# 安装构建依赖...
```

或者使用其他镜像源：
- 阿里云：`mirrors.aliyun.com`
- 清华大学：`mirrors.tuna.tsinghua.edu.cn`
- 中科大：`mirrors.ustc.edu.cn`

### 解决方案 2：使用代理

如果服务器有代理，可以设置环境变量：

```bash
export HTTP_PROXY=http://your-proxy:port
export HTTPS_PROXY=http://your-proxy:port
docker-compose -f docker-compose.prod.yml up -d --build
```

### 解决方案 3：增加超时时间

如果网络较慢但稳定，可以增加构建超时时间（在 docker-compose.yml 中不适用，但可以在 Dockerfile 中优化）。

### 解决方案 4：分步构建

如果网络不稳定，可以分步构建：

```bash
# 1. 先构建基础镜像（只安装系统依赖）
docker build -t diary-backend-base -f backend/Dockerfile.base backend/

# 2. 再构建完整镜像
docker-compose -f docker-compose.prod.yml build
```

## 快速修复（推荐）

如果服务器在国内，直接修改 Dockerfile：

```bash
# 在服务器上执行
cd MyShards-Diary
sed -i 's/dl-cdn.alpinelinux.org/mirrors.aliyun.com/g' backend/Dockerfile
docker-compose -f docker-compose.prod.yml up -d --build
```

或者手动编辑 `backend/Dockerfile`，取消注释镜像源那一行。

## 验证网络速度

测试镜像源速度：

```bash
# 测试默认源
time apk update

# 测试阿里云源（临时）
sed -i 's/dl-cdn.alpinelinux.org/mirrors.aliyun.com/g' /etc/apk/repositories
time apk update
```

## 构建时间参考

- **正常网络**：5-10 分钟
- **国内服务器 + 镜像源**：2-5 分钟
- **慢速网络**：15-30 分钟
- **超时**：如果超过 30 分钟，建议使用镜像源或代理

---

**建议**：如果服务器在国内，务必使用国内镜像源，可以大幅提升构建速度。
