# 测试镜像源可用性

在正式构建之前，建议先测试镜像源是否可用，避免构建几个小时然后失败。

## 方法 1：使用测试脚本（推荐）

在服务器上执行：

```bash
# 1. 进入项目目录
cd MyShards-Diary

# 2. 给脚本添加执行权限
chmod +x test-mirror.sh

# 3. 运行测试（需要在 Alpine 容器内运行）
docker run --rm -it node:20-alpine sh -c "
  apk add --no-cache curl
  curl -I http://mirrors.aliyun.com/alpine/v3.19/main/x86_64/APKINDEX.tar.gz
"
```

如果返回 200 OK，说明镜像源可用。

## 方法 2：使用测试 Dockerfile

```bash
# 1. 构建测试镜像（只测试镜像源，不安装所有依赖）
docker build -f backend/Dockerfile.test -t diary-test backend/

# 2. 如果构建成功（几秒钟内完成），说明镜像源可用
# 3. 如果失败或超时，说明镜像源有问题
```

## 方法 3：快速测试（最简单）

在服务器上直接测试：

```bash
# 测试阿里云镜像
docker run --rm node:20-alpine sh -c "
  sed -i 's/dl-cdn.alpinelinux.org/mirrors.aliyun.com/g' /etc/apk/repositories && \
  timeout 30 apk update && \
  echo '✅ 镜像源可用'
"
```

如果 30 秒内完成并显示 "✅ 镜像源可用"，说明镜像源正常。

## 方法 4：测试网络连接

```bash
# 测试能否访问镜像源
curl -I http://mirrors.aliyun.com/alpine/v3.19/main/x86_64/APKINDEX.tar.gz
# 或
wget --spider http://mirrors.aliyun.com/alpine/v3.19/main/x86_64/APKINDEX.tar.gz
```

## 如果所有镜像源都不可用

1. **检查网络连接**：
   ```bash
   ping mirrors.aliyun.com
   ```

2. **检查防火墙**：
   ```bash
   # 确保 80 和 443 端口开放
   ```

3. **使用代理**：
   ```bash
   export HTTP_PROXY=http://your-proxy:port
   export HTTPS_PROXY=http://your-proxy:port
   ```

4. **使用默认源**（如果服务器在国外）：
   编辑 `backend/Dockerfile`，注释掉镜像源那一行。

## 推荐测试流程

1. **先测试镜像源**（使用方法 3，最快）：
   ```bash
   docker run --rm node:20-alpine sh -c "sed -i 's/dl-cdn.alpinelinux.org/mirrors.aliyun.com/g' /etc/apk/repositories && timeout 30 apk update && echo '✅ 可用'"
   ```

2. **如果测试通过，再正式构建**：
   ```bash
   git pull
   docker-compose -f docker-compose.prod.yml up -d --build
   ```

3. **如果测试失败，尝试其他镜像源**：
   编辑 `backend/Dockerfile`，修改镜像源地址。

---

**建议**：每次部署前都先测试镜像源，避免浪费时间。
