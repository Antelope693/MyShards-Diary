# 📤 推送代码到 GitHub 和服务器部署指南

## 🔐 第一步：推送代码到 GitHub

代码已提交到本地仓库，现在需要推送到 GitHub。

### 如果推送时遇到认证问题

**方法一：使用 Personal Access Token（推荐）**

1. 访问 GitHub：https://github.com/settings/tokens
2. 点击 "Generate new token" → "Generate new token (classic)"
3. 填写 Note（如：MyShards Deployment）
4. 勾选 `repo` 权限
5. 点击 "Generate token"
6. **复制生成的 token**（只显示一次！）

推送时：
```bash
git push -u origin main
# Username: 输入你的 GitHub 用户名
# Password: 输入刚才复制的 token（不是密码！）
```

**方法二：使用 SSH**

1. 生成 SSH 密钥（如果还没有）：
   ```bash
   ssh-keygen -t ed25519 -C "your_email@example.com"
   ```

2. 复制公钥：
   ```bash
   cat ~/.ssh/id_ed25519.pub
   ```

3. 添加到 GitHub：
   - GitHub → Settings → SSH and GPG keys → New SSH key
   - 粘贴公钥内容

4. 更改远程地址：
   ```bash
   git remote set-url origin git@github.com:Antelope693/MyShards-Diary.git
   ```

5. 推送：
   ```bash
   git push -u origin main
   ```

## 🚀 第二步：服务器部署

### 前提条件

- ✅ 代码已推送到 GitHub
- ✅ 有 Linux 服务器
- ✅ 服务器已安装 Docker 和 Docker Compose

### 快速部署步骤

#### 1. 连接到服务器

```bash
ssh root@你的服务器IP
```

#### 2. 安装 Docker（如果未安装）

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh
systemctl start docker
systemctl enable docker

# 安装 Docker Compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

#### 3. 克隆项目

```bash
cd /opt
git clone https://github.com/Antelope693/MyShards-Diary.git diary
cd diary
```

#### 4. 创建配置文件

```bash
# 复制配置模板
cp env.example .env

# 编辑配置文件
nano .env
```

**必须修改的内容：**

```env
# 管理员密码（必须修改！）
ADMIN_PASSWORD=你的强密码

# 服务器域名或IP（必须填写！）
SERVER_DOMAIN=你的域名或IP
```

保存：`Ctrl + O` → `Enter` → `Ctrl + X`

#### 5. 部署

```bash
# 方法一：使用部署脚本
chmod +x deploy.sh
./deploy.sh

# 方法二：手动部署
docker-compose -f docker-compose.prod.yml up -d --build
```

#### 6. 检查状态

```bash
# 查看服务状态
docker-compose -f docker-compose.prod.yml ps

# 查看日志
docker-compose -f docker-compose.prod.yml logs -f
```

#### 7. 配置防火墙

```bash
# Ubuntu/Debian
sudo ufw allow 80/tcp
sudo ufw enable

# CentOS
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --reload
```

#### 8. 访问网站

- 前端：`http://你的域名` 或 `http://你的IP`
- 管理：`http://你的域名/admin` 或 `http://你的IP/admin`

当代码更新后，在服务器上：

```bash
cd /opt/diary
git pull
docker-compose -f docker-compose.prod.yml up -d --build
```

## 🆘 遇到问题？

1. **推送失败**：检查 GitHub 认证方式
2. **部署失败**：查看日志 `docker-compose -f docker-compose.prod.yml logs`
3. **无法访问**：检查防火墙和端口
4. **登录失败**：检查 `.env` 文件中的密码

更多帮助请查看 [完整部署指南.md](./完整部署指南.md) 的"常见问题"部分。
