# 创建 .env 文件说明

## 问题

使用 `docker-compose.prod.yml` 时出现警告：
```
The "ADMIN_PASSWORD" variable is not set. Defaulting to a blank string.
```

这是因为缺少 `.env` 配置文件。

## 解决方法

### 方法一：手动创建 .env 文件（推荐）

1. **复制配置模板**
   ```bash
   copy env.example .env
   ```
   或
   ```bash
   cp env.example .env
   ```

2. **编辑配置文件**
   
   Windows:
   ```bash
   notepad .env
   ```
   
   Linux/Mac:
   ```bash
   nano .env
   ```

3. **填写必要配置**

   至少需要填写：
   ```env
   ADMIN_PASSWORD=你的密码
   SERVER_DOMAIN=localhost
   ```

   完整示例（本地开发）：
   ```env
   ADMIN_PASSWORD=admin123
   SERVER_DOMAIN=localhost
   FRONTEND_PORT=80
   BACKEND_PORT=3001
   NODE_ENV=production
   TZ=Asia/Shanghai
   ```

4. **保存文件**

5. **重启服务**
   ```bash
   docker-compose -f docker-compose.prod.yml down
   docker-compose -f docker-compose.prod.yml up -d
   ```

### 方法二：使用开发环境配置（本地测试推荐）

如果只是本地测试，可以使用开发环境配置，不需要 .env 文件：

```bash
# 停止生产环境
docker-compose -f docker-compose.prod.yml down

# 启动开发环境（端口 3000）
docker-compose up -d --build
```

然后访问：http://localhost:3000
