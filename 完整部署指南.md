## 什么是 .env 文件？

### 🤔 .env 文件是什么？

`.env` 文件是一个**配置文件**，用来存储你的应用设置。你可以把它想象成应用的"设置面板"。

### 📝 为什么需要 .env 文件？

- **安全性**：密码等敏感信息不会写在代码里
- **灵活性**：不同服务器可以用不同配置，无需修改代码
- **简单性**：只需要填写一次，以后都自动使用

### 🔍 .env 文件长什么样？

`.env` 文件就是普通的文本文件，里面是一行行的配置，格式如下：

```
配置项名称=配置的值
```

例如：
```
ADMIN_PASSWORD=MyPassword123
SERVER_DOMAIN=diary.example.com
```

### ⚠️ 重要提示

- `.env` 文件包含敏感信息，**不要**上传到公开的代码仓库
- 项目中的 `.env.example` 是**模板文件**，你需要复制它并填写自己的信息

## 服务器准备

### 1. 购买/准备服务器

你需要一台 Linux 服务器，推荐配置：
- **操作系统**：Ubuntu 20.04 或更高版本（推荐）
- **内存**：至少 2GB
- **硬盘**：至少 10GB 可用空间
- **网络**：有公网 IP 地址

### 2. 连接到服务器

#### Windows 用户

1. 下载 **PuTTY** 或使用 **Windows Terminal**
2. 打开终端，输入：
   ```bash
   ssh root@你的服务器IP
   ```
   例如：`ssh root@192.168.1.100`
3. 输入服务器密码（输入时不会显示，直接输入后按回车）

#### Mac/Linux 用户

1. 打开终端
2. 输入：
   ```bash
   ssh root@你的服务器IP
   ```
3. 输入服务器密码

### 3. 更新系统

连接成功后，先更新系统：

```bash
# Ubuntu/Debian
apt update && apt upgrade -y

# CentOS
yum update -y
```

## 安装 Docker

Docker 是用来运行应用的容器工具。我们需要先安装它。

### Ubuntu/Debian 系统

```bash
# 1. 安装必要的工具
apt install -y curl

# 2. 下载并安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# 3. 启动 Docker 服务
systemctl start docker
systemctl enable docker

# 4. 将当前用户添加到 docker 组（可选，避免每次都用 sudo）
usermod -aG docker $USER

# 5. 安装 Docker Compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# 6. 验证安装
docker --version
docker-compose --version
```

如果看到版本号，说明安装成功！

### CentOS 系统

```bash
# 1. 安装必要的工具
yum install -y curl

# 2. 下载并安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sh get-docker.sh

# 3. 启动 Docker 服务
systemctl start docker
systemctl enable docker

# 4. 将当前用户添加到 docker 组
usermod -aG docker $USER

# 5. 安装 Docker Compose
curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose

# 6. 验证安装
docker --version
docker-compose --version
```

## 上传项目文件

### 方法一：使用 Git（推荐）

如果你使用 Git 管理代码：

```bash
# 1. 安装 Git（如果还没有）
apt install -y git  # Ubuntu/Debian
# 或
yum install -y git  # CentOS

# 2. 克隆项目
cd /opt
git clone 你的项目地址 diary
cd diary
```

### 方法二：使用 SCP 上传

#### Windows 用户

1. 下载 **WinSCP** 或使用 PowerShell
2. 在 PowerShell 中执行：
   ```powershell
   scp -r D:\work\Diary root@你的服务器IP:/opt/diary
   ```

#### Mac/Linux 用户

```bash
scp -r /path/to/Diary root@你的服务器IP:/opt/diary
```

然后在服务器上：

```bash
cd /opt/diary
```

### 方法三：使用 FTP 工具

1. 使用 FileZilla 等 FTP 工具
2. 连接到服务器
3. 上传整个项目文件夹到 `/opt/diary`

## 配置环境变量

### 1. 创建 .env 文件

```bash
# 进入项目目录
cd /opt/diary

# 复制配置模板
cp env.example .env
```

### 2. 编辑 .env 文件

```bash
# 使用 nano 编辑器（简单易用）
nano .env
```

### 3. 填写配置信息

在打开的编辑器中，你会看到类似这样的内容：

```env
# 管理员密码（必须修改！）
ADMIN_PASSWORD=your_secure_password_here

# 服务器域名或IP
SERVER_DOMAIN=your-domain.com

# 端口配置（默认即可）
FRONTEND_PORT=80
BACKEND_PORT=3001
```

**修改说明：**

1. **ADMIN_PASSWORD**（必须修改）
   - 这是你登录管理页面的密码
   - 请设置一个强密码，例如：`MySecurePassword123!`
   - 修改示例：`ADMIN_PASSWORD=MySecurePassword123!`

2. **SERVER_DOMAIN**（必须修改）
   - 如果你有域名：填写域名，例如：`diary.example.com`
   - 如果只有IP：填写服务器IP，例如：`192.168.1.100`
   - 修改示例：`SERVER_DOMAIN=diary.example.com`

3. **FRONTEND_PORT**（通常不需要修改）
   - 前端访问端口，默认 80
   - 如果 80 端口被占用，可以改为其他端口，如 `8080`

4. **BACKEND_PORT**（通常不需要修改）
   - 后端API端口，默认 3001

### 4. 保存文件

在 nano 编辑器中：
- 按 `Ctrl + O` 保存
- 按 `Enter` 确认文件名
- 按 `Ctrl + X` 退出

### 5. 验证配置

```bash
# 查看配置文件内容（确认已正确填写）
cat .env
```

确保 `ADMIN_PASSWORD` 和 `SERVER_DOMAIN` 不再是默认值！

## 部署应用

### 方法一：使用部署脚本（推荐）

```bash
# 给脚本添加执行权限
chmod +x deploy.sh

# 运行部署脚本
./deploy.sh
```

脚本会自动：
1. 检查配置文件
2. 停止旧容器（如果有）
3. 构建 Docker 镜像
4. 启动服务
5. 检查服务状态

### 方法二：手动部署

```bash
# 构建并启动服务
docker-compose -f docker-compose.prod.yml up -d --build
```

### 部署过程说明

部署可能需要 5-10 分钟，你会看到类似这样的输出：

```
Building backend...
Building frontend...
Creating diary-backend ...
Creating diary-frontend ...
```

这是正常的，请耐心等待。

## 验证部署

### 1. 检查服务状态

```bash
docker-compose -f docker-compose.prod.yml ps
```

应该看到两个服务都是 `Up` 状态：

```
NAME              STATUS
diary-backend     Up
diary-frontend    Up
```

### 2. 查看日志

```bash
# 查看所有服务日志
docker-compose -f docker-compose.prod.yml logs -f
```

按 `Ctrl + C` 退出日志查看。

如果没有错误信息，说明部署成功！

在浏览器中访问：

- **前端地址**：`http://你的域名` 或 `http://你的IP`
- **管理页面**：`http://你的域名/admin` 或 `http://你的IP/admin`

### 4. 测试登录

1. 访问管理页面
2. 输入你在 `.env` 文件中设置的 `ADMIN_PASSWORD`
3. 如果成功登录，说明一切正常！

## 配置防火墙

如果无法访问网站，可能是防火墙阻止了端口。

### Ubuntu/Debian (UFW)

```bash
# 允许 80 端口（HTTP）
sudo ufw allow 80/tcp

# 如果使用 HTTPS，允许 443 端口
sudo ufw allow 443/tcp

# 启用防火墙
sudo ufw enable

# 查看防火墙状态
sudo ufw status
```

### CentOS (firewalld)

```bash
# 允许 HTTP
sudo firewall-cmd --permanent --add-service=http

# 允许 HTTPS（如果使用）
sudo firewall-cmd --permanent --add-service=https

# 重新加载防火墙
sudo firewall-cmd --reload

# 查看防火墙状态
sudo firewall-cmd --list-all
```

## 常见问题

### ❌ 问题1：无法访问网站

**可能原因：**
1. 防火墙未开放端口
2. 服务未正常启动
3. 域名未正确解析

**解决方法：**
```bash
# 1. 检查服务状态
docker-compose -f docker-compose.prod.yml ps

# 2. 检查日志
docker-compose -f docker-compose.prod.yml logs

# 3. 检查端口是否被占用
netstat -tulpn | grep :80

# 4. 检查防火墙
sudo ufw status  # Ubuntu
sudo firewall-cmd --list-all  # CentOS
```

### ❌ 问题2：登录失败

**可能原因：**
1. 密码错误
2. `.env` 文件中的密码未正确设置

**解决方法：**
```bash
# 检查 .env 文件
cat .env | grep ADMIN_PASSWORD

# 如果密码不对，重新编辑
nano .env

# 修改后重启服务
docker-compose -f docker-compose.prod.yml restart backend
```

### ❌ 问题3：Docker 命令找不到

**解决方法：**
```bash
# 重新加载用户组（如果添加了用户到 docker 组）
newgrp docker

# 或使用 sudo
sudo docker-compose -f docker-compose.prod.yml ps
```

### ❌ 问题4：端口被占用

**解决方法：**
```bash
# 查看占用端口的进程
sudo lsof -i :80

# 停止占用端口的服务，或修改 .env 中的端口
nano .env
# 修改 FRONTEND_PORT=8080
```

### ❌ 问题5：构建失败

**解决方法：**
```bash
# 清理旧的镜像和容器
docker system prune -a

# 重新构建
docker-compose -f docker-compose.prod.yml build --no-cache
```
